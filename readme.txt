(2)提升流程品質或系統品質的措施:
1.code review :
	-code至少有2個人看過
	-code可讀性
	-註解 
	-變數命名有沒有意義
	-code功能是否正常，符合預期
	-code至少要有1個人看得懂
2.安全性:
	-https協議加密
	-AES演算法加密個資
3.定義資料庫schema:
	-Primary key
	-required
	-Not Null
	-資料型態
4.merge或遇到conflict時，舉行線上會議討論
5.git commit message規範:
	-feat: 新增功能
	-modify:修改功能
	-fix:修復bug
	-style:網頁格式以及code格式修改
6.branch命名規定: 
	- branch開發者名稱 + 開發日期 +時間　ex: Jyun 11/21 18:42
	- 一個人最多兩個分支，一個最新的正在改的，一個上一版的
	-一個功能就commit一次
